Задание 5 — Поиск в динамических множествах

Существует большое количество алгоритмов, предназначенных для эффективного поиска элементов в динамических множествах: бинарные деревья поиска, различные способы их балансировки, хеш-таблицы и различные методы решения коллизий. Разные алгоритмы подходят для разных ситуаций — в зависимости от характера данных, распределения значений, выполняемых операций и т.д. В этом задании ваша задача — реализовать и сравнить эти подходы.

В рамках задания необходимо реализовать и затем сравнить 3 реализации структур данных с точки зрения эффективности операций insert, delete и search в различных ситуациях:

(2 балла) Собственная реализация бинарного дерева поиска (BST) с любым алгоритмом балансировки, например:
AVL-дерево, Splay-дерево, Красно-чёрное дерево, (2,3)-дерево, (2,3,4)-дерево, весово-балансированные деревья и т.п.

(2 балла) Вторая собственная реализация BST с другим алгоритмом балансировки, чем в предыдущем пункте.

(4 балла) Собственная реализация хеш-таблицы с:

цепочками (chaining)

двойным хешированием (double hashing)
Также необходимо реализовать динамическое изменение размера хеш-таблицы.

За сами реализации можно получить максимум 8 баллов.

Каждую реализацию нужно сдать в отдельном исходном файле.
Если сдаёте все четыре реализации — значит будет 4 файла.

Запрещено использовать чужой код.

Для успешной сдачи нужно реализовать как минимум две структуры из перечисленных (только хеш-таблица — недостаточно).

Правильность нужно проверить тестированием и сравнением с другими реализациями.

Техническая документация

В документации нужно:

описать все реализованные структуры данных

указать подробные сценарии тестирования

показать, в каких ситуациях какая структура работает эффективнее

Эффективность оценивается как:

среднее время выполнения одной операции в зависимости от количества элементов.

Также необходимо сдать программу для тестирования и измерения эффективности (отдельный файл с функцией main).

⚠️ Без тестовой программы и сравнения минимум двух реализаций работа оценивается в 0 баллов.

За документацию, тестирование и результаты можно получить до 6 баллов

Оценивается:

качество тестирования

анализ результатов в документации

Документация должна содержать:

таблицы

графики

где сравнивается производительность (скорость) различных решений.

Так как результаты зависят не только от реализации, но и от сценариев тестирования (последовательностей операций insert / delete / search), важно протестировать разные сценарии.
